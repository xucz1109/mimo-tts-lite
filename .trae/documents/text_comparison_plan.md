# MiMo TTS Lite - 文本对比功能实现计划

## 项目调研总结

**当前项目状态**:
- 单页面应用，所有功能都在 `public/index.html` 中
- 使用原生JavaScript，没有框架依赖
- 已有文本输入、语音生成、历史记录、文本校验功能
- 使用CSS变量定义样式主题

**技术栈确认**:
- HTML5 + CSS3 + JavaScript (ES6+)
- Express.js 后端
- 浏览器 localStorage

**存在的约束与风险**:
- 需要处理中文文本对比的复杂性（标点、换行、空格）
- 性能：大文本对比可能影响页面响应
- 需要保持与现有UI风格一致

## 技术方案设计

**整体架构**:
在现有单页面应用中添加一个新的tab页，通过切换显示不同功能区域。文本对比功能完全在前端实现，不需要后端支持。

**核心模块划分**:
1. **Tab切换模块**
   - 负责切换不同功能页（TTS、文本对比）
2. **文本对比模块**
   - 文本预处理（去除标点、空白字符）
   - 文本对比算法（基于字符级别的差异对比）
   - 可视化渲染（绿色标注相同，黄色标注不同）
3. **UI渲染模块**
   - 左右分栏布局
   - 实时对比显示

**目录结构规划**:
不新增目录，所有代码都在现有文件中实现

## 潜在风险与注意事项

**技术风险**:
- 大文本对比性能问题 → 缓解措施: 添加文本长度限制和防抖处理
- 标点符号处理复杂性 → 缓解措施: 定义明确的标点符号集合

**依赖风险**:
- 无新增依赖，风险较低

**操作风险**:
- 修改现有HTML结构可能影响现有功能 → 缓解措施: 保守修改，保持现有功能不变

## 验证方案

**功能验证**:

1. **Tab切换功能**
   - 操作: 点击不同的tab按钮
   - 预期: 页面内容正确切换，样式正常

2. **基本文本对比**
   - 操作: 在左右两侧输入相同文本
   - 预期: 所有文本显示为绿色

3. **差异文本对比**
   - 操作: 在左右两侧输入部分不同的文本
   - 预期: 不同部分显示为黄色，相同部分显示为绿色

4. **跨行对比**
   - 操作: 输入包含换行的文本进行对比
   - 预期: 正确处理换行，对比结果准确

5. **标点符号处理**
   - 操作: 左侧输入带标点的文本，右侧输入不带标点的文本
   - 预期: 能够正确识别为相同文本
